UI SHOUT

A browser-native DAW. No install. No account. No limits on ideas.

What is this

UI SHOUT is a full digital audio workstation that lives in a single HTML file. It runs entirely in your browser using the Web Audio API — no server, no plugins, no electron wrapper. Open the file, make music.

It was built for people who think in Logic Pro X but want something they can open on any machine, share with a URL, or hack directly.

Getting started

Transport

  • Press Space to play/stop. Enter rewinds to zero.
  • Set BPM in the top toolbar number field.
  • Click the timeline ruler to move the playhead.
  • Toggle Cycle to loop between the yellow markers.

Tracks

  • Click + Track to add a track. Choose MIDI, Bass, Pad, Drummer, or Audio.
  • Drag the ⠿ handle to reorder tracks.
  • Click a track header to select it — its instrument loads in the Inspector panel.
  • M = mute, S = solo. Vol and pan sliders are on each track header.

Regions

  • Double-click an empty lane to create a region.
  • Drag a region to move it. Drag the right edge to resize.
  • Double-click a region to open it in the Piano Roll.
  • Right-click a region for: duplicate, rename, colour, mute, delete.
  • With S (scissors tool) active, click a region to split it at that beat.

Views

📐 Arrange

The main timeline. All your tracks and regions live here. Drag regions, resize them, open them in the piano roll.

🎚 Mixer

One channel per track. Volume fader, pan knob, EQ, reverb, delay sends. The master bus has a limiter.

🥁 Step Sequencer

16-step grid for drums and rhythm parts. Each row is a different sound. Syncs to transport BPM. Independent play button or locked to main transport.

Piano Roll

  • Click in Draw mode to add a note. Drag to set length.
  • Switch to Select mode to drag, resize, or delete notes.
  • Quantize, transpose, and octave shift via the toolbar inside the roll.
  • Notes are stored per-region — each region has its own sequence.

Inspector (I)

Per-track instrument settings: wave shape, ADSR envelope, filter frequency, resonance, distortion drive, reverb send, delay send, detune.

Changes are live — you hear them immediately during playback.

Recording

  • Click 🎙 Input to configure your mic input.
  • Arm a track (🔴 button on track header), then press R or the rec button.
  • Punch in/out markers set the record region. Recorded audio appears as a waveform region.

Browser-only perks unique

🔗 Share

Encodes your entire project (MIDI regions, settings, BPM) into a URL. Paste it anywhere — recipient opens it and the project loads automatically. No server involved.

🌐 Tab Rec

Captures audio from any browser tab — YouTube, Spotify, a reference track, a synth plugin running in another tab. Routes it live into a new track. Completely impossible in a native DAW without a virtual audio device.

☀ Wake

Requests a screen wake lock so your display stays on during a session. Releases automatically when the tab hides.

〰 Scope

Live oscilloscope on the master output. Tap it on mid-session to see what the signal is doing.

🎹 MIDI Export

Exports a standard type-1 .mid file — one MIDI track per project track, correct tempo, all note-on/off events and velocities. Opens in every DAW, notation app, and hardware synth on earth.

Saving

Three backends — choose on first launch or via the ⋯ button:

  • Browser storage — fast, no permissions, ~5MB limit. Fine for MIDI-only sessions. Audio regions may not survive reload for large sessions.
  • Local file — saves a real .uishout JSON file to your hard drive. Unlimited size. Audio regions preserved as base64 PCM. Requires Chrome or Edge 86+.
  • None — no auto-save. Use ⬇ Export WAV to keep your work.

Auto-save runs every 60 seconds when browser storage is active. File storage saves on every manual 💾 Save press.

MIDI control

Open the MIDI panel to map any hardware CC knob or fader to master volume, BPM, track volume, pan, filter, reverb, delay, or transport. Click Learn, wiggle the hardware control, done.

UI SHOUT

Made in a browser. Runs in a browser. Ships as one file.

✦ The idea

Most DAWs are heavy. They ask you to install, account, activate, configure. UI SHOUT asks nothing — you open a file and it works. It was designed in conversation, built iteratively, and lives entirely in ~3800 lines of HTML.

The name is the sound a UI makes when it finally does what you actually want.

Tech stack

  • Web Audio API — all synthesis, mixing, effects, recording, export
  • Canvas 2D — piano roll, waveform previews, oscilloscope, step sequencer
  • Web MIDI API — hardware controller mapping
  • File System Access API — native hard drive save/load (Chrome/Edge)
  • getDisplayMedia — tab audio capture
  • Screen Wake Lock API — display stays on during sessions
  • Zero dependencies — no framework, no bundler, no node_modules

What's inside

  • Full multi-track arrange view with regions, colours, labels
  • Piano roll with draw, select, quantize, transpose, velocity
  • Step sequencer — 16 sounds × 32 steps, synced to transport
  • 6 synth types: MIDI, Bass, Pad, Pluck, Piano, Drummer
  • Per-track: filter, reverb send, delay send, distortion, detune, ADSR
  • Shared reverb convolver + feedback delay bus
  • Mixer view with per-track faders, pan, sends
  • Automation lanes (volume, pan, filter, reverb)
  • Mic recording with punch in/out, comping takes
  • Undo/redo with full snapshot history
  • MIDI hardware CC mapping with learn mode
  • WAV export via OfflineAudioContext
  • MIDI file export (type-1 .mid)
  • Project sharing via URL hash encoding
  • In-app runtime error logger

Limitations (honest)

  • Browser storage ~5MB — audio regions need File storage backend
  • Tab Rec requires Chrome or Edge (getDisplayMedia with audio)
  • MIDI file export needs Chrome or Edge 86+ for File System access
  • No VST/AU plugins — synthesis is all native Web Audio nodes
  • No multicore rendering — OfflineAudioContext is single-threaded

✦ Future ideas

Collaborative sessions via WebRTC. AudioWorklet-based custom synths. Import MIDI files. Spectral view. Mobile touch support. A version that fits in a tweet.

All of them possible. None of them requiring an installer.
